# 第一篇碎碎念

![第一张演示图片](images/fulls/01.jpg)

这是一个示例文件。

以后你只需要：

1. 新建一个 `.md` 文件
2. 丢进 `blog/posts/`
3. 推到 GitHub

页面就会自动识别它，并在 **碎碎念** 页面里显示出来。

## 为什么要这样做

把写作入口保留成最简单的 `.md`，会比每次都手动改网页舒服很多。你只要专心写内容，页面本身会自动完成生成、渲染和上线。

![第二张演示图片](images/fulls/05.jpg)

> 风格会保持成简约、理性、偏包豪斯的排版。
